# Agent instructions (Cursor + Claude Code)

このリポジトリは **Cursor**（`.cursor/`）と **Claude Code**（`CLAUDE.md` / `.claude/`）を併用する前提で整理されています。どちらのツールでも、ここに書いた方針を最優先してください。

## 優先順位

1. ユーザーの明示的な指示
2. このファイルおよび `CLAUDE.md` の内容（内容が重なる場合は同じ方針を優先）
3. `.cursor/rules/*.mdc`（Cursor 向けの補足・パターン別ルール）
4. 言語・フレームワークの一般的ベストプラクティス

## コミュニケーション

- ユーザーは日本語でのやり取りを好む。説明・要約・コミットメッセージの説明は日本語でよい。
- コード中の識別子・コメント方針はプロジェクトの既存スタイルに合わせる（未確定なら英語コメントでも可）。

## 作業姿勢

- 依頼のスコープ以外のリファクタや無関係ファイルの編集をしない。
- 変更理由を短く明確に伝える。大きな変更の前は方針を一言確認する。
- 秘密情報（API キー、トークン、`.env`）をコードやログに含めない。

## プロジェクト構成（設定ファイル）

| 用途 | 場所 |
|------|------|
| エージェント共通の方針 | `AGENTS.md`（本ファイル）、`CLAUDE.md` |
| Cursor ルール | `.cursor/rules/*.mdc` |
| Cursor スキル | `.cursor/skills/<name>/SKILL.md` |
| Claude Code 設定 | `.claude/settings.json`（共有）、`.claude/settings.local.json`（個人・コミットしない） |

## スキル（Cursor）

プロジェクト固有ワークフローは `.cursor/skills/` を参照すること。コミットメッセージ、レビュー、ツール切り替え時の引き継ぎなどを定義している。

## スタック未確定時

ランタイムやフレームワークがまだない場合は、導入時に `README.md` と本ファイルに「実行・ビルド・テストコマンド」を追記する。
